✅ Episode Panel Visual Issues - Fixed

Issue: Broken CSS/Visual Glitches When Panel Opens
Symptoms:
  • Pie charts sliding away unexpectedly
  • Blurred background showing broken layout
  • Visual artifacts during panel animation
Root Causes:
  • Backdrop blur filter causing rendering issues with transforming content
  • Content push animation conflicting with blur effect
  • Z-index stacking context problems
Fix 1: Removed Backdrop Blur
Before (Broken)
backdrop-filter: blur(8px);
-webkit-backdrop-filter: blur(8px);
After (Fixed)
/* Removed backdrop-filter */
background: rgba(26, 26, 46, 0.5);
Why: Backdrop blur can cause rendering issues when applied to content that's being transformed or has complex z-index stacking.
Fix 2: Changed Content Push to Scale Transform
Before (Margin Push)
margin-right: 700px;
After (Scale Transform)
transform: scale(0.95) translateX(-2%);
transform-origin: left center;
Why: Scale transforms are more performant and less likely to cause layout reflow issues than margin changes.
Fix 3: Added Stacking Context Fixes
body.panel-open .pie-chart,
body.panel-open .pie-charts-container,
body.panel-open svg {
  transform: translateZ(0);
  backface-visibility: hidden;
}

Why: Creates proper stacking contexts for SVG elements to prevent them from appearing in wrong z-order.
Fix 4: Improved Animation Sequencing
Changed the order of operations:
  1. Add panel-open class to body first
  2. Use requestAnimationFrame to let layout stabilize
  3. Then show backdrop and panel
Why: Gives the browser time to properly calculate layout before starting animations.
✨ Result
The episode panel should now:
  • Open smoothly without visual glitches
  • Display a clean backdrop without blur artifacts
  • Scale the main content gracefully on larger screens
  • Maintain proper z-index ordering for all elements
